# Implementation Plan: Crash Triage Overhaul + Cmplog + Coverage Improvements

## Overview

Three implementation phases, ordered by impact:

1. **Phase 1: Crash triage overhaul** — Enrich crash output with full context for faster triage
2. **Phase 2: Cmplog (comparison tracing)** — LD_PRELOAD shim intercepting memcmp/strcmp/etc, feeding operands into dictionary + mutations
3. **Phase 3: Coverage improvements** — Sanitizer LD_PRELOAD shim, per-seed edge bitmask, redqueen, differential fuzzing, AFL import

### Changes

#### 1.1 CrashMetadata dataclass (`core/crash_metadata.py`)

New dataclass collecting all triage context:

```python
@dataclass
class CrashMetadata:
    # From sanitizer report
    sanitizer: str
    error_type: str
    fault_addr: str
    frames: list[str]
    access_size: int | None = None  # READ of size N / WRITE of size N
    access_type: str | None = None  # "READ" / "WRITE" / "FREE"
    shadow_info: str | None = None  # full shadow memory description
    alloc_stack: list[str] | None = None
    dealloc_stack: list[str] | None = None

    # Exploitability
    exploitability: str = "UNKNOWN"  # CRITICAL / HIGH / MEDIUM / LOW

    # Cluster ID (short hash of signature)
    cluster_id: str = ""

    # Execution metadata
    timestamp: str = ""
    fuzzer_pid: int = 0
    exec_count: int = 0
    corpus_size: int = 0
    parent_seed_hash: str = ""
    mutation_ops: list[str] = field(default_factory=list)
    target: str = ""
    target_sha256: str = ""
    elapsed: str = ""

    # Crash input analysis
    input_hexdump: str = ""
    input_text_repr: str = ""
    nearest_corpus_file: str = ""
    nearest_similarity: float = 0.0
    diff_bytes: list[int] = field(default_factory=list)

    # Register state (if ptrace active)
    rip: int = 0
    rsp: int = 0
    rbp: int = 0
    instruction_bytes: str = ""
```

#### 1.2 Enrich SanitizerReport (`core/sanitizer.py`)

Add new regex patterns and fields:
- `SANITIZER_ACCESS_RE` — matches `READ of size N` / `WRITE of size N` / `FREE of size N`
- `SANITIZER_SHADOW_RE` — matches shadow memory description lines
- `SANITIZER_ALLOC_RE` — matches `allocated by thread T0 here` + frames
- `SANITIZER_DEALLOC_RE` — matches `freed by thread T0 here` + frames
- New `__slots__` fields: `access_size`, `access_type`, `shadow_info`, `alloc_frames`, `dealloc_frames`
- Exploitability lookup table as class method: `ASAN_EXPLOITABILITY`

#### 1.3 Enrich `save_crash` (`adapters/filesystem.py`)

New signature:
```python
def save_crash(
    data: bytes,
    returncode: int,
    stderr: str,
    crashes_dir: Path,
    crash_hashes: set[str],
    crash_sigs: dict[str, int],
    metadata: CrashMetadata | None = None,  # NEW
) -> bool:
```

New output files per crash:
1. `crash_{timestamp}_{cluster_id}_{error_short}.bin` — crash input (renamed from hash-only)
2. `crash_{timestamp}_{cluster_id}_{error_short}.txt` — enriched sidecar with all metadata
3. `crash_{timestamp}_{cluster_id}_{error_short}.sh` — reproducer script (base64 input + ASAN_OPTIONS)
4. `crash_{timestamp}_{cluster_id}_{error_short}.hex` — hexdump of input

The cluster ID is `hashlib.sha256(sig.encode()).hexdigest()[:8]`.

#### 1.4 Pass context from Fuzzer (`services/fuzzer.py`)

Update `Fuzzer.save_crash()` to build `CrashMetadata` from:
- `self.exec_count`, `self.corpus`, `self.seed_meta`, `self._last_ops_used`
- `self.target`, `self.start_time`
- Compute `target_sha256` once in `__init__`
- Find nearest corpus entry by byte-level Jaccard on 4-grams
- If `self.ptrace_cov` is active, capture register state from the last SIGTRAP

#### 1.5 Reproducer script generation

Template:
```bash
#!/bin/bash
# Reproducer: {error_type} @ {top_frame}
# Generated: {timestamp}
# Input SHA256: {input_hash}
echo '{base64_input}' | base64 -d | \
  ASAN_OPTIONS=abort_on_error=1:symbolize=1:detect_leaks=0 \
  {target}
```

## Phase 2: Cmplog (Comparison Tracing)

### Goal
Intercept comparison functions via LD_PRELOAD, log operand pairs, feed them into the dictionary and mutation engine.

### Files to modify
- `src/fuzzer_tool/adapters/shim_factory.py` — add `build_cmplog_shim()`
- `src/fuzzer_tool/core/mutations.py` — add cmplog-aware mutations
- `src/fuzzer_tool/services/fuzzer.py` — wire cmplog into execution path

### New files
- `adapters/cmplog_shim.c` — C source for LD_PRELOAD shim
- `core/cmplog.py` — `CmplogCollector` class

### Changes

#### 2.1 C shim (`adapters/cmplog_shim.c`)

Intercepts via `dlsym(RTLD_NEXT, ...)`:
- `memcmp(a, b, n)` — log both operands when result != 0
- `strcmp(a, b)` — log both strings
- `strncmp(a, b, n)` — log both strings up to n
- `memchr(haystack, needle, n)` — log haystack and needle

Communication: write operand pairs to a shared file (`_CMPLOG_OUT` env var), format:
```
CMP <offset_in_input> <operand_hex> <cmp_result>\n
```

#### 2.2 CmplogCollector (`core/cmplog.py`)

```python
class CmplogCollector:

        self.log_path = log_path
        self.input_data = input_data

    def read_log(self) -> list[bytes]:
        """Parse CMP lines, extract operand bytes, return as tokens."""
        ...

    def correlate_with_input(self, operand: bytes) -> int | None:
        """Find where operand appears in the input. Returns offset or None."""
        ...
```

#### 2.3 Integration

- `build_cmplog_shim()` in `shim_factory.py` — compile `cmplog_shim.c` to `.so`
- In `Fuzzer._run_target()`: set `LD_PRELOAD=<cmplog_shim>`, `_CMPLOG_OUT=<path>`
- After each execution: `CmplogCollector.read_log()` → feed tokens into `self.dictionary` and `self.mc.cem` elite set
- Cmplog tokens get priority weight in dictionary mutations (10x regular dict weight)

## Phase 3: Coverage Improvements

### 3.1 Sanitizer coverage LD_PRELOAD shim

New file: `adapters/sancov_shim.c`

Intercepts `__sanitizer_cov_trace_pc_guard(guard)` and writes edge index to a shared bitmap file. Gives coverage feedback for any Clang-compiled binary without AFL instrumentation.

Integration: `build_sancov_shim()` in `shim_factory.py`, set `LD_PRELOAD` + `_COV_BITMAP_OUT` in env.

### 3.2 Per-seed edge bitmask

Add `edge_bitmap: set[int] | None` to seed metadata. After each execution that produces new coverage, record which edges are now hit. During `_pick_seed()`, deprioritize seeds whose coverage is fully subsumed by others.

### 3.3 Redqueen-style input-to-state

When an input reaches a new edge, record which input bytes were at the branch comparison point (from cmplog). On subsequent mutations of that seed, prefer flipping those specific bytes.

Implementation: `RedqueenTracker` class that maps `(seed, edge) → byte_offsets`.

### 3.4 Differential fuzzing mode

Add `--differential TARGET_B` flag. For each input, run both targets and compare outputs (returncode + stdout). Flag divergence as a finding (not necessarily a crash).

New subcommand or flag on `fuzzer-tool fuzz`.

### 3.5 Corpus import from AFL/libFuzzer

New subcommand: `fuzzer-tool import --from-afl <afl_out_dir>`

Reads `queue/id:*` files and `crashes/crash-*` files, copies them into fuzzer corpus/crashes directories with proper naming.

## Implementation Order

1. **Phase 1** (Crash triage) — 3-4 files, self-contained, immediate triage value
2. **Phase 2.1-2.3** (Cmplog) — C shim + Python collector + wiring
3. **Phase 3.5** (AFL import) — Simple CLI addition
4. **Phase 3.1** (Sancov LD_PRELOAD shim) — Similar to cmplog shim
5. **Phase 3.2-3.3** (Seed edge bitmask + redqueen) — Build on cmplog
6. **Phase 3.4** (Differential) — Independent, can be parallelized
